I want to implement a mobile companion mode, not a responsive version of the desktop app.

Before building anything new, remove or disable any existing mobile-specific or responsive code that was added during previous attempts to make the full app mobile compatible. This includes:

Mobile CSS overrides

Hover disabling hacks

Responsive layout changes

Mobile nav experiments
The goal is to reset mobile behaviour back to a clean baseline before introducing the companion mode.

Core principle

Desktop behaviour, layout, styling, and interactions must remain completely unchanged

Mobile is a separate, stripped-down experience, not a resized desktop

Treat mobile as its own interface with its own navigation and layout

When to activate mobile companion mode

On mobile screen sizes, load the mobile companion interface instead of the desktop UI

Do not attempt to adapt or reflow desktop components for mobile

Mobile companion goals

The mobile companion should be designed for quick, practical use while out and about, not deep configuration.

It should prioritise speed, clarity, and tap-friendly interactions.
